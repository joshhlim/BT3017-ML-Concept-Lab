The Kernel Trick
How SVMs classify non-linearly separable data by lifting it into higher dimensions
1 / 4

SVMs and the Separation Problem

A Support Vector Machine (SVM) is a classifier that finds the hyperplane separating two classes with the maximum margin — the widest possible gap between the boundary and the nearest points from each class. Wider margin = better generalisation to unseen data.

The points sitting exactly on the margin boundary are called support vectors — they are the only training points that determine the boundary. Every other point could be removed without changing the result.

But SVMs assume data is linearly separable. Real data often isn't — imagine red dots in a ring around blue dots. No straight line can split them. This is the problem the kernel trick solves: it lets SVMs learn curved, non-linear boundaries without any extra computational cost.

Linear SVM: f(x) = w·x + b
Maximise: margin = 2 / ‖w‖
Subject to: yᵢ(w·xᵢ + b) ≥ 1 ∀i

❌ Fails when no hyperplane separates the classes
✓ Kernel trick fixes this — next steps show how
The kernel trick in one sentence: Instead of computing in the original space, the kernel trick implicitly maps data into a higher-dimensional space where it becomes linearly separable — without ever computing the mapping explicitly.
Non-linearly separable data — no line works here

The Lifting Idea

The key insight: if we can't separate the data in its current space, lift it into a higher-dimensional space where it becomes separable. This transformation is called a feature map, denoted φ(x).

Consider our circles. In 2D, they overlap when projected onto any line. But if we add a third dimension equal to the radius r² = x₁² + x₂², the inner ring sits at low z-values and the outer ring at high z-values. A flat horizontal plane now separates them perfectly!

This is the geometric intuition: what appears entangled in low dimensions often untangles beautifully in higher dimensions. The SVM then finds a hyperplane in this lifted space — which corresponds to a curved boundary back in the original space.

Feature map: φ: ℝ² → ℝ³
(x₁, x₂) ↦ (x₁, x₂, x₁² + x₂²)

In lifted space: separating plane at z = threshold
In original space: this is a circle!
Problem ahead: For rich feature maps (e.g. all degree-3 terms in 100D), φ(x) has millions of dimensions — too expensive to compute explicitly. We need a smarter approach.
Lifted into 3D — circles become planes

The Kernel Trick — What It Actually Does

The SVM can be rewritten in its dual form, where the optimisation only ever involves pairs of training points through their inner products φ(xᵢ)·φ(xⱼ) — never φ(x) alone. This is the key observation that makes the trick possible.

Define a kernel function K(x, x') = φ(x)·φ(x'). If we can compute this inner product directly from x and x' — without computing φ at all — we get all the benefits of the high-dimensional feature space for free. This is the kernel trick: replace every inner product in the SVM with K(x,x').

For the RBF kernel K(x,x') = exp(−γ‖x−x'‖²), the implied φ maps into an infinite-dimensional space, yet computing K takes only O(d) time. We are doing infinite-dimensional geometry in linear time.

SVM dual uses only: φ(xᵢ)·φ(xⱼ)
→ Replace with: K(xᵢ, xⱼ) ← the trick!

RBF: K(x,x') = exp(−γ‖x−x'‖²) → ∞-dim φ
Poly: K(x,x') = (x·x' + c)ᵈ → dᵈ-dim φ
Linear: K(x,x') = x·x' → same space

✓ Never compute φ(x) explicitly — K does it all
Mercer's theorem: Any positive semi-definite function K(x,x') is a valid inner product in some feature space. We don't need to know what φ is — K encodes everything needed to train and classify.
RBF kernel: K(x,x') — similar points score ≈1, distant points ≈0

The Decision Boundary

After training with a kernel, the SVM decision function in input space becomes curved — shaped entirely by the kernel geometry. Points close to support vectors (in kernel space) are attracted to one side; far points to the other.

The γ parameter of the RBF kernel controls the scale of the similarity. Large γ: only very nearby points count as similar → tight, jagged boundaries (risk of overfitting). Small γ: distant points still have influence → smoother, more global boundaries.

The C parameter controls the margin softness. High C: enforce correct classification at all costs (small margin). Low C: allow some misclassifications to get a wider, more generalising margin.

Decision: f(x) = Σᵢ αᵢ yᵢ K(xᵢ, x) + b
Only support vectors have αᵢ > 0

Large γ → narrow "influence radius" per point
Small γ → wide "influence radius" → smoother

What PCA Does — The Full Picture

Principal Component Analysis (PCA) is an unsupervised technique that compresses high-dimensional data by finding the directions of greatest spread. Given data with d features, PCA produces a new set of axes — the principal components — ranked by how much variance they capture.

Here is what PCA does, step by step: (1) Centre the data (subtract the mean of each feature). (2) Optionally standardise (divide by std dev) so no feature dominates due to scale. (3) Compute the covariance matrix Σ. (4) Find its eigenvectors — these are the principal components. (5) Project the data onto the top-k eigenvectors to get a k-dimensional representation.

The core idea: project data onto a unit vector u. The variance of the projected values equals uᵀΣu. PCA finds the u that maximises this — the direction of greatest spread.

Step 1: centre X̃ = X − μ
Step 2: covariance Σ = (1/n) X̃ᵀX̃
Step 3: eigendecompose Σ = V Λ Vᵀ
Step 4: project Z = X̃ Vₖ (keep top k cols)

Result: Z is n×k — compressed representation
Key property: The principal components are orthogonal — they capture completely independent directions of variation. PCA simultaneously compresses data AND removes correlations between features.
Two projection directions — the diagonal captures far more variance

The Covariance Matrix and Eigenvectors

The covariance matrix Σ is a d×d matrix where entry (i,j) measures how features i and j vary together. Diagonal entries are per-feature variances. Off-diagonal entries reveal correlations: if height and weight tend to increase together, their covariance is positive.

PCA solves the variance-maximisation problem by computing the eigendecomposition of Σ. Each eigenvector vᵢ is a principal component — a direction in feature space. Its corresponding eigenvalue λᵢ tells you exactly how much variance the data has in that direction. The eigenvectors are always orthogonal, so the principal components are always uncorrelated.

In the plot, the arrows show PC1 (longest — most variance) and PC2 (shorter, perpendicular). Rotating the axes to align with these arrows is what PCA does: it finds a new coordinate system where the axes point along the data's natural directions of spread.

Covariance matrix: Σ = (1/n) X̃ᵀX̃
Eigendecomposition: Σ vᵢ = λᵢ vᵢ

λ₁ ≥ λ₂ ≥ ... ≥ λ_d ≥ 0
v₁·v₂ = 0 (orthogonal — uncorrelated)

Variance in direction vᵢ = λᵢ
Total variance = Σ λᵢ = trace(Σ)
Why eigenvectors? For any direction u, variance = uᵀΣu. This is maximised when u is the top eigenvector of Σ — a fundamental result from linear algebra called the Rayleigh quotient theorem.
PC arrows — length ∝ √eigenvalue (variance in that direction)

Projection and Reconstruction

Once we have the principal components, we can use them to compress data. We project each point onto the top-k components to get a k-dimensional representation. This is the encoding step. To get back an approximation of the original data, we project back — this is the decoding step.

The difference between the original and reconstructed points is the reconstruction error. Every component we drop contributes to this error by an amount equal to its eigenvalue. So if we keep components that together explain 95% of total variance, we lose only 5% — a huge compression with minimal information loss.

In the Sandbox, you can see this live: with a 5D dataset and only 2 components retained, the reconstruction in 2D still captures most of the structure because the data has strong correlations between features.

Encode: z = Vₖᵀ (x − μ) (n×d → n×k)
Decode: x̂ = Vₖ z + μ (n×k → n×d)

Reconstruction error = ‖x − x̂‖²
= Σᵢ>ₖ λᵢ (sum of dropped eigenvalues)

Important: standardise first! (z-score each feature)
Trade-off: More components → less error, less compression. The optimal k depends on your downstream task and acceptable information loss.
Reconstruction error — lines show information lost

Choosing k — The Scree Plot

How many components should we keep? The scree plot plots eigenvalues (or explained variance ratios) from largest to smallest. Look for the "elbow" — the point where adding another component gives diminishing returns.

A more principled approach is the eigengap heuristic: choose k where there is the largest jump (gap) between consecutive eigenvalues λₖ and λₖ₊₁. A big gap means PC k captures something structurally different from the noise-level components beyond it.

In practice, people often target a cumulative explained variance threshold — e.g., keep enough components to explain 95% of total variance. In the Sandbox you can watch this threshold update live as you drag the component slider.

Explained ratio: rᵢ = λᵢ / Σⱼ λⱼ
Cumulative: Rₖ = Σᵢ₌₁ᵏ rᵢ

Common thresholds: 90%, 95%, 99%
Eigengap rule: choose k where λₖ/λₖ₊₁ is largest

For correlated 5D data, often 1–2 PCs suffice!

Spectral Clustering
Graph-based clustering that finds non-convex structure by operating on eigenvectors of the Laplacian

Why K-Means Fails

K-means is a great algorithm, but it has a fundamental geometric limitation: it partitions the space into Voronoi regions — convex regions defined by which centroid is nearest. This means k-means can only find clusters that are roughly convex (blob-shaped).

Consider two concentric circles. Their centroids coincide at the origin. K-means, assigning each point to its nearest centroid, will draw a vertical or diagonal line — mixing inner and outer ring points in both clusters. It has no concept of "this point is topologically surrounded by its own group."

The same failure occurs with interlocking crescents (moons), spirals, or any dataset where clusters are non-convex but locally connected. We need an algorithm that respects topology and connectivity, not just Euclidean distance to a centroid.

K-means objective:
min_{C₁,...,Cₖ} Σₖ Σᵢ∈Cₖ ‖xᵢ − μₖ‖²

Assigns each point to its nearest centroid μₖ
→ Only works for convex, roughly spherical clusters
→ Fails for rings, spirals, moons, manifolds
The fix: Instead of asking "which centroid is nearest?", ask "which points are connected to me through a path of similar neighbours?" This is the graph-theoretic view.
K-means splits the rings incorrectly (vertical boundary)

The Affinity Graph

The first step of spectral clustering builds a similarity graph. Each data point becomes a node. Edge weights encode how similar two points are — high weight for nearby points, near-zero for distant ones.

The most common choice is the RBF (Gaussian) kernel as the affinity function. Points within the same ring will have high mutual similarity weights. Points on different rings are far apart and get near-zero weights — they're essentially disconnected in the graph.

The parameter γ controls how quickly similarity decays with distance. Large γ: only very close points are considered similar (sparse, local connections). Small γ: more distant points still have some similarity (denser graph). Getting γ right is crucial — in the Sandbox you can tune it live.

Affinity: Wᵢⱼ = exp(−γ ‖xᵢ − xⱼ‖²)
(Set Wᵢᵢ = 0, i.e. no self-loops)

Large γ: only near neighbours connected
Small γ: wider neighbourhood, denser graph

Alternative: k-nearest-neighbour graph (k-NN)
Key intuition: Points in the same ring are locally connected through chains of nearby neighbours. Points across rings are globally far. The graph captures this; raw distance to a centroid doesn't.
Affinity matrix — bright = similar, dark = distant

The Graph Laplacian

From the affinity matrix W, we construct the graph Laplacian L = D − W, where D is the diagonal degree matrix (Dᵢᵢ = Σⱼ Wᵢⱼ). The Laplacian is the central mathematical object: its eigenvectors reveal cluster structure.

There's a deep theorem here: the number of connected components in the graph equals the number of zero eigenvalues of L. For two perfectly disconnected rings, L has exactly two zero eigenvalues. In practice, clusters are noisy, so we look at the eigengap — the largest jump between consecutive eigenvalues tells us k.

In practice we use the normalised Laplacian Lₛ = D⁻½ L D⁻½ = I − D⁻½ W D⁻½. This corrects for degree heterogeneity and produces much better clusters.

Unnormalised: L = D − W
Normalised: Lₛ = D⁻½ L D⁻½ = I − D⁻½ W D⁻½

Theorem: rank(L) = n − (# connected components)
→ # zero eigenvalues = # components

Eigengap heuristic: choose k = argmax (λₖ₊₁ − λₖ)
Reading the spectrum: If you see eigenvalues [0.00, 0.01, 0.54, 0.88, ...], the gap after the second value tells you k=2. In the Sandbox, the eigenvalue bar chart highlights this gap.
Eigenvalue spectrum — eigengap reveals k=2

Spectral Embedding + K-Means

The final step assembles the algorithm. We stack the first k eigenvectors of Lₛ as columns: U = [u₁ | u₂ | ... | uₖ]. Each row of U is a k-dimensional spectral embedding of the corresponding data point.

The magic: in this embedding space, points that are graph-connected (in the same cluster) map to similar locations, while disconnected points are far apart — regardless of their positions in the original space. The two concentric rings, which overlap in 2D, become two well-separated clouds in the spectral embedding!

We then row-normalise U (so each point lies on the unit sphere) and apply standard k-means. Because the spectral embedding has already done the hard work, k-means easily finds clean partitions. The whole algorithm runs in O(n² + n·k) with sparse approximations.

1. Compute affinity W (RBF kernel)
2. Compute Lₛ = D⁻½(D−W)D⁻½
3. Find k smallest eigenvectors of Lₛ → U ∈ ℝⁿˣᵏ
4. Row-normalise: Ũᵢ = Uᵢ / ‖Uᵢ‖
5. K-means on rows of Ũ

Complexity: O(n²) affinity + O(n²k) eigendecomposition

Graph Neural Networks
Learn node representations by aggregating information from neighbours — message passing in action

Why Graphs Matter

Much of the world's most interesting data is structured as graphs: social networks (users and friendships), molecules (atoms and bonds), citation networks (papers and references), road maps, protein interaction networks, and more.

Standard deep learning architectures — CNNs, RNNs, MLPs — are designed for grids (images), sequences (text), or fixed-size vectors. They break down on graphs because: (1) graphs have variable structure — no two graphs have the same number of nodes or edges; (2) there is no natural node ordering — permuting node labels shouldn't change the model's output (permutation invariance).

The core insight of GNNs: a node's identity and role in the graph can be learned by looking at its neighbourhood. A person in a social network is partly defined by who they know. An atom in a molecule is partly defined by what atoms it's bonded to. GNNs formalise this idea.

Graph G = (V, E)
V: nodes (with features xᵢ ∈ ℝᵈ)
E: edges (connections, can be weighted)
A ∈ {0,1}ⁿˣⁿ: adjacency matrix

Goal: learn embedding hᵢ for each node i
that captures features AND graph structure
Key requirement: The model must be permutation-invariant — renaming nodes A→B→C or changing their order shouldn't change predictions. Standard MLPs fail this; GNNs satisfy it by design.
Graph with node features (bars inside each node)

Message Passing

The core operation in GNNs is message passing. It has three steps that run simultaneously for all nodes:

1. Message: Each node i computes a message to send to each of its neighbours. The simplest message is just the node's current feature vector.

2. Aggregate: Each node collects all incoming messages from its neighbours and combines them — typically by taking their sum, mean, or max. This is the neighbourhood aggregation step.

3. Update: Each node combines its aggregated neighbourhood information with its own current representation, passing it through a learned transformation (a small neural network layer) to produce its new embedding.

This process repeats for L layers. After L rounds, each node's embedding has "seen" information from all nodes within L hops — its L-hop neighbourhood.

Message: mⱼ→ᵢ = φ(hⱼ) (e.g. hⱼ itself)
Aggregate: aᵢ = AGG({mⱼ→ᵢ : j ∈ N(i)})
Update: hᵢ' = σ(W · CONCAT(hᵢ, aᵢ))

L layers → each node sees L-hop neighbourhood
Receptive field grows by 1 hop per layer
Analogy: Think of rumour spreading. After 1 round, you've heard from your direct friends. After 2 rounds, you've heard from friends-of-friends. GNNs propagate information the same way.
Dashed arrows show messages flowing to the highlighted node

The Matrix Form (GCN)

The Graph Convolutional Network (GCN) by Kipf & Welling (2017) is the most widely used GNN variant. It implements message passing as a single matrix operation across all nodes simultaneously — efficient and elegant.

The key matrix is Â = D̂⁻¹(A + I): the adjacency matrix with added self-loops (so each node includes itself), normalised by degree (so aggregation is a weighted average rather than a sum, preventing scale issues for high-degree nodes).

Multiplying by Â effectively replaces each node's features with a weighted average of its own features and its neighbours' features. The learnable weight matrix W then transforms this aggregated representation. After applying a non-linearity σ (e.g. ReLU), we have the next layer's embeddings.

GCN update rule:
H⁽ˡ⁺¹⁾ = σ( Â H⁽ˡ⁾ W⁽ˡ⁾ )

Â = D̂⁻¹(A + I) (normalised adj. + self-loops)
H⁽ˡ⁾ ∈ ℝⁿˣᵈˡ: node features at layer l
W⁽ˡ⁾ ∈ ℝᵈˡˣᵈˡ⁺¹: learnable weights
σ: activation (ReLU, tanh, etc.)
Why Â H is clever: (Â H)ᵢ is a weighted average of node i and all its neighbours' feature vectors. W learns what linear combination of these averaged features to extract. The graph structure and the learning are cleanly separated.
Â encodes which nodes average with whom (small matrix shown)

Tasks and Applications

GNNs can be applied at three different levels of the graph, each corresponding to different real-world tasks:

Node-level: Predict a label for each node using its final embedding. Example: classify users in a social network as bots vs. humans, predict which paper belongs to which research field.

Edge-level: Predict properties of pairs of nodes — e.g. whether an edge should exist between them. Example: recommend friends, predict drug-drug interactions, knowledge graph completion.

Graph-level: Aggregate all node embeddings into a single graph-level representation and predict a label for the whole graph. Example: predict whether a molecule is toxic, classify a circuit as functional, identify graph isomorphism.

For graph-level tasks, a readout function aggregates node embeddings: typically global sum, mean, or max pooling, or more sophisticated hierarchical pooling (DiffPool, MinCutPool).

Node classification: y_i = f(h_i)
Link prediction: ŷ_ij = g(h_i, h_j) e.g. h_i · h_j
Graph classification: ŷ_G = MLP(READOUT({h_i}))

READOUT options: Σᵢhᵢ, (1/n)Σᵢhᵢ, maxᵢhᵢ,
or learned hierarchical pooling
